Netty 3.6.0.Beta1 - And more to come

by normanmaurer
on 12-Nov-2012

And another release.... 

Netty 3.6.0.Beta1 ships all the fixes which are are included in Netty 3.5.10.Final , but contains one extra enhancement which we felt would be to "heavy" to have it included in the 3.5.x series.

In Netty 3.6.0.Beta1 we fixed one long standing known limitation that was often a critic of our users. And to be honest they are right, so it was time to fix it

In the past when you used Netty in Server mode with the NIO-Transport it was needed to have one Thread allocated per bound port. That's not a big deal if you only bind to a handful of ports, but can be just not "feasible" if you plan to bind to thousands of ports.

So we changed this to allow to share the same Thread for many bound ports. This is done for you automatically without any change needed by you. But it goes further, as you can also specify how many Threads you want use. One is just the default which may work out for you or not, as it depends on the workload.

To support this we added two new constructors to the NioServerChannelFactory

For more details see [#721].

The JAR file can be found on Maven Central or  you can also get the full tarball from the download page. Whatever you prefer...

Please let us know if you have any problems or questions.
